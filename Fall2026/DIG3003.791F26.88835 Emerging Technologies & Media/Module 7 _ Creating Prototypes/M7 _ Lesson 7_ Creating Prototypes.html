Lesson 6: Creating Prototypes

Wireframe, Mockup, & Prototype

 Lecture

By now, it is clear that a satisfying user experience and an effective user interface do not pop out of thin air. We need to test the design alternatives with users. Using a prototype is a useful way to test the design alternative.

However, prototypes are also not siphoned out of the vacuum either. Creating wireframes and mockups often precedes coming up with a test-ready prototype. The following video explains what wireframe, mockup, and prototype are and how they differ. There are two important learning points from the video:

  1. The main take-home message here is that it is easier and more efficient to develop a low-fidelity sketch first rather than running straight into developing a prototype. Distinguishing the minute difference between wireframe, mockup, or prototype is not as important as the video suggests. The difference between mockups and prototypes, especially, is often vague. What's more important is that we want acceptable visual fidelity and interactivity for testing.
  2. The video does a good job of introducing some tools for prototyping and how/what they can be used for. See this page for more information about the tools that can be used for prototyping.
