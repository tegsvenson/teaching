IDEA 9 | Reporting the Results

Here, you will create a document that analyzes and summarizes your findings from IDEA 7.

Draw the Analysis

You should have the screen recording and the very brief interview with the user from IDEA 7. With the data in hand, write the report by filling out this analysis template: Usability Testing Report.docx

In addition to answering the questions in the template, report:

  • The average values from each data point across the two participants for the two interfaces that you tested. Report all the indicators and answers to the subjective questions for your usability goal or design principle.
  • Clearly refer to the recordings you submitted using the file names you used for IDEA 7 at least once, and define which files represent which session. If you have submitted files that include multiple sessions, refer to the time stamps in the file as well.
  • You may quote the part of your interview as evidence during your analysis.

Submission

  • Provide a written analysis by answering the following questions:
    • Provide a brief description about the user (age, gender, first name or alias, your relationship with them) without sharing too much personal identifiers.
    • Was the user behavior (mouse clicks, keystrokes, steps, errors, time, etc.) different or similar between the apps? Which one has larger/smaller numbers?
    • Was the task outcome (price) different between the apps? Which one has larger/smaller numbers?
    • What was the user response for the three questions? By comparing it to the other numbers (behaviors and outcome), which do you think are correlated and contributed to user satisfaction?
    • What is your thoughts on the overall implication that you can infer from the data that you collected?
  • Up to 1500 words including the template. 
  • DOCX or PDF
