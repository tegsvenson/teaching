A Mac or Windows computer is required to complete this assignment. This course assumes and requires that students use a Mac or Windows computer as their main device for accessing course content. In this assignment, we practice a systematic approach to measuring user experience. It is easy to claim that an app is user-friendly or not. These claims only become meaningful with adequate support. We are creating supporting evidence in this exercise. 

Note. This assignment is NOT part of the project assignments. However, you are practicing how to conduct an analysis that you will later do in the project assignment when you test your prototype's usability. You will replicate the data collection method in this assignment as part of your project.

 Assignment Instructions

Assignment Objective

Later in this course, you will test the prototype that you have developed. At that point, you will collect data. We are trying out the data collection process in this assignment. 

Data of Interest

In the earlier lessons, we learned that a user-friendly interface lets users achieve more with less effort or error. Less effort can be translated to: 

  • fewer mouse clicks
  • fewer keystrokes
  • fewer steps to complete a task
  • fewer occurrences of going back and forth due to errors or changing decisions
  • less time to complete a task
  • better results (better price or product) 
  • ask the users about their experience by asking questions (see below)

Usability Test

We will run a very simple Usability Test. We are comparing two different applications or websites on a computer. Here is the exercise we will do: 

  • Find two users to execute the task for you: this can be anyone (friend, family member, etc.), but make sure they are capable of finishing the task without getting help from others.
  • Meeting in person will make this assignment go easier.
  • Run screen recording, including mouse clicks and keyboard input.
    • How to record screen with mouse clicks on MacOS and Windows.
    • You may use an alternative tool of your choice, but it must be a screen capture and not a camera footage of the screen.
  • Have your user go through their task while you are recording the screen.
  • After the user finishes their task, conduct a brief interview with the questions below.
  • Write a report based on your findings on user activity and the brief interview.

The Task for your user :

  • Purchasing a round-trip plane ticket from Tampa to Paris, starting on December 23 and returning on January 2
  • Choose two different websites for purchasing plane tickets
    • Expedia, Priceline, etc
    • For the first website: Have your user go through the steps for purchasing the plane ticket, right before the point where the user "Place the Order"
    • Repeat the task on the second website. 

After the user finishes going through the steps on both sites, ask and record the following three questions. 

  • Which website worked better for you?
  • Why do you prefer that website over the other one?
  • Which website was easier to use? 

Now, you should have the screen recording and the very brief interview with the user. With the data in hand, write the report by filling out this analysis template.    Quantitative Analysis Assignment Template

The Report

  • Provide a written analysis by answering the following questions:
    • Provide a brief description about the user (age, gender, first name or alias, your relationship with them) without sharing too much personal identifiers.
    • Was the user behavior (mouse clicks, keystrokes, steps, errors, time, etc.) different or similar between the apps? Which one has larger/smaller numbers?
    • Was the task outcome (price) different between the apps? Which one has larger/smaller numbers?
    • What was the user response for the three questions? By comparing it to the other numbers (behaviors and outcome), which do you think are correlated and contributed to user satisfaction?
    • What is your thoughts on the overall implication that you can infer from the data that you collected?
  • Up to 1500 words including the template.
